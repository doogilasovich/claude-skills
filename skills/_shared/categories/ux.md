# UX Review Categories

10 review/fix specialist pairs for user experience.

## Categories

| ID | Key | Review Focus | Dev Focus |
|----|-----|--------------|-----------|
| 1 | friction | Journey obstacles, tap counts, hesitation | Error handling, loading states |
| 2 | delight | Animations, haptics, celebrations | Micro-interactions |
| 3 | accessibility | VoiceOver, Dynamic Type, contrast | Labels, scaling, colors |
| 4 | monetization | Paywall timing, value communication | Paywall optimization |
| 5 | retention | Habit loops, streaks, triggers | Notifications, achievements |
| 6 | virality | Share friction, social proof, loops | Share flows, referrals |
| 7 | onboarding | First-run, time-to-value, permissions | Progressive disclosure |
| 8 | trust | Privacy, security perception, errors | Transparency, controls |
| 9 | cognitive-load | Info density, decisions, memory req | Simplification, defaults |
| 10 | platform-compliance | HIG, native patterns, safe areas | Native components |

## Skill Naming

- Review: `ux-{category}` suffix (e.g., `ux-friction-analyst`)
- Fix: `dev-{category}` prefix (e.g., `dev-friction-fixer`)
